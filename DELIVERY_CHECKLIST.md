## 🎓 關鍵實現細節

### 1. 代客建檔 + 圖片上傳

**流程：**
```
1. Admin 填寫表單（車行、車型、年份、售價）
2. 選擇圖片（可選，支援拖放）
3. 點擊「建立並上架」
4. 前端驗證表單
5. POST /admin/vehicles/proxy （建立車輛）
6. 如有圖片，POST /admin/vehicles/{id}/images （上傳）
7. 顯示成功提示，重定向到 /dashboard
```

**亮點：**
- 先建立再上傳（分離關注點）
- 自動處理失敗情況
- 進度提示改善體驗

### 2. 審核流程

**流程：**
```
1. Admin 訪問 /admin/audit
2. 查看待審核車輛列表
3. 點擊車輛進入詳情
4. 選擇核准或拒絕
5. 確認操作
6. 返回列表，自動刷新
```

**亮點：**
- 完整的狀態轉移
- 拒絕原因記錄
- 操作日誌追蹤
- 列表自動刷新

### 3. 批量圖片上傳

**技術：**
- Multer `upload.array('images', 10)`
- 逐個驗證：格式、大小
- 詳細摘要：成功數、失敗數、原因

## 📞 技術支援

### 常見問題解答

**Q: 如何測試圖片上傳？**  
A: 參考 `VERIFICATION_GUIDE.md` 中的「場景 1: 代客建檔 + 圖片上傳」

**Q: 為什麼拒絕後看不到車輛？**  
A: 拒絕後車輛狀態變為 'rejected'，不再顯示在待審核列表中

**Q: 如何查看審核日誌？**  
A: 檢查數據庫中的 audit_logs 表

**Q: 支援多少語言的拒絕原因？**  
A: 支援任何語言（UTF-8），無長度限制
